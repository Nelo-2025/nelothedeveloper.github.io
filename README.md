# StayTech: Digital Infrastructure for Local Stay N' Play Communities

---

StayTech is a comprehensive frontend solution designed to bridge the gap between local parents/caregivers and community Stay N' Play groups. Working as a volunteer with Stay N' Play group (early childhood development) discovered parents and group problem with discovering events and booking. This web app is dedicated to increase visibility, showcase events and scale. The platform is structured as a **scalable B2B2C SaaS model** with clear routes to monetization.

### The Business Opportunity 

Rather than operating purely as a community directory, StayTech acts as a scalable hub addressing critical operational pain points for community centers, local councils, and corporate sponsors:

- **Freemium Marketplace (B2B):** Free for parents to discover local events; premium tier unlocks advanced community tools and personalized AL-driven event recommendations.
  
- **SaaS Analytics Dashboard (B2B):** Local authorities and early childhood educators pay a subscription fee to access **aggregated engagement data collection, analysis, and reporting services** to audit community impact and optimize funding allocation.
  
- **Automated Bookings & Admin Levers:** Mitigates operational overhead for local organizers through a turnkey booking, check-in, and capacity management system.


### Frontend Technical Architecture & Skills Demonstration

This project exhibits clean, modular, and semantic frontend engineering practices designed to maximize accessibility and responsiveness.

### Core Architecture

1. **Semantic HTML5 & Accessibility:** Implemented modern, SEO-friendly structural elements (`<header>`, `<main>`, `<section>`, `<nav>`) to ensure screen-reader compatibility and seamless search indexability.

2. **Advanced Responsive Layouts:** Crafted custom CSS architectures (utilizing CSS Grid and Flexbox layouts) ensuring a mobile-first experience so parents can easily book events while on the move.

3. **Dynamic Client-Side Interactions:** Built robust vanilla javaScript controllers (`admin.js`, `parent,js`, `listevents.js`) to handle multi-role conditional rendering, real-time category filtering, and mock session state management.

### Key Functional Portals Developed:

1. **Public Discovery Engine (`index.html`, `listevents.html`):** Features client-side filtering, allowing users to sort real-time listings by categories like *Story Time, Playdate, Crafts, and Music*

2. **Parent Hub (`Parent.html`):** An intuitive personal dashboard featuring unified login state changes, target booking metrics, and tracking UI

3. **Admin Center (`admin.html`):** A protected layout interface showcasing specialized metrics, role-restricted messaging, and quick-links to event population managers.
